# Étape 8: PR (Pull Request)

**Objectif**: Créer une Pull Request avec le bon format Izzico.

## Processus de Création de PR

### 8.1 Vérifications Préalables

```bash
# Vérifier le status git
git status

# Vérifier la branche actuelle
git branch --show-current

# Vérifier les commits non poussés
git log origin/main..HEAD --oneline
```

### 8.2 Créer une Branche (si nécessaire)

Si sur `main`:
```bash
# Créer une branche feature
git checkout -b feature/[feature-name-kebab-case]
```

Conventions de nommage:
- `feature/[nom]` - Nouvelle fonctionnalité
- `fix/[nom]` - Correction de bug
- `refactor/[nom]` - Refactoring
- `docs/[nom]` - Documentation

### 8.3 Commit des Changements

**Format du message de commit**:
```
[type]: [description courte]

[description détaillée si nécessaire]

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>
```

Types:
- `feat`: Nouvelle fonctionnalité
- `fix`: Correction de bug
- `refactor`: Refactoring
- `style`: Changements de style/design
- `docs`: Documentation
- `test`: Ajout/modification de tests

### 8.4 Push vers Remote

```bash
git push -u origin [branch-name]
```

### 8.5 Créer la Pull Request

Utilise `gh pr create` avec le format Izzico:

```bash
gh pr create --title "[type]: [description]" --body "$(cat <<'EOF'
## Summary

[2-3 bullet points décrivant les changements]

## Changes

### Files Created
- `[path/file.tsx]` - [description]

### Files Modified
- `[path/file.tsx]` - [changements]

## Acceptance Criteria Verified

- [x] [Critère 1]
- [x] [Critère 2]
- [x] [Critère 3]

## Reviews Completed

| Review | Status | Issues |
|--------|--------|--------|
| Design V3-fun | ✅ | 0 |
| RGPD/Security | ✅ | 0 |
| Voice Guidelines | ✅ | 0 |

## Test Plan

- [ ] [Test manuel 1]
- [ ] [Test manuel 2]
- [ ] [Test automatisé: `npm run test`]

## Screenshots (if UI changes)

[Ajouter screenshots si applicable]

## Notes

[Notes additionnelles pour les reviewers]

---
Generated with Claude Opus 4.5 via Izzico Feature Workflow
EOF
)"
```

### 8.6 Rapport Final

```
★ Pull Request Created ───────────────────────────

Feature: [FEATURE_DESCRIPTION]
Branch: [branch-name]
PR URL: [url]

Stats:
- Files changed: X
- Lines added: +Y
- Lines removed: -Z

Reviews Passed:
- ✅ Design V3-fun
- ✅ RGPD/Security
- ✅ Voice Guidelines

Tests: [PASS | SKIPPED | N/A]

Next Steps:
1. Review the PR at [url]
2. Request human review if needed
3. Merge when approved

──────────────────────────────────────────────────
```

## Workflow Terminé

Le workflow Izzico Feature est maintenant terminé.

**Résumé des étapes complétées**:
1. ✅ ANALYZE - Contexte compris
2. ✅ PLAN - Plan établi
3. ✅ EXECUTE - Implémentation faite
4. ✅ VALIDATE - Critères vérifiés
5. ✅ EXAMINE - Self-review complète (si -X)
6. ✅ RESOLVE - Issues corrigées (si nécessaire)
7. ✅ TEST - Tests passés (si -T)
8. ✅ PR - Pull Request créée (si -PR)

**La feature est prête pour review humaine.**
